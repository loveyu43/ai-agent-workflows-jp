# Examples

These examples show how operators can use the templates without granting an AI agent uncontrolled write access.

## Included Examples

- [Chatwork support triage](chatwork-support-triage-example.md)
- [MCP provider catalog validation](mcp-provider-catalog-validation-example.md)

## Operating Rule

Use read-only access first. Treat writes, external sends, customer communication, accounting changes, HR changes, contract changes, server operations, DNS, SSL, database changes, and paid API consumption as human-approved operations.
