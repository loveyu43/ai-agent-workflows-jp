# AIエージェント業務承認境界マトリクス

AIエージェントに業務を任せるときは、MCPかAPIかよりも「何を実行できる権限を持っているか」を先に確認します。

## 境界レベル

| レベル | 例 | 自動実行 | 承認 | ログ |
| --- | --- | --- | --- | --- |
| L0 読み取り | 検索、一覧取得、要約、棚卸し | 可 | 初回設定時 | 取得元と時刻 |
| L1 下書き | 返信案、記事構成、仕訳候補、タスク案 | 可 | 公開・登録前 | 入力と出力 |
| L2 内部更新 | kintone更新、Backlog課題追加、esa WIP投稿 | 原則不可 | 毎回 | 差分、対象ID、件数 |
| L3 外部送信 | 顧客返信、LINE通知、請求書送信、営業メール | 不可 | 毎回 | 宛先、本文、承認者 |
| L4 高リスク実行 | 支払い、契約、労務、人事、税務、RPA、本番サーバー変更 | 不可 | 担当者または専門家 | 証跡、根拠、ロールバック |
| L5 破壊的操作 | 削除、権限変更、DB変更、DNS/SSL変更、監視停止 | 不可 | 明示承認 + バックアップ | 事前/事後証跡 |

## 業務別の推奨初期設定

| 業務 | 推奨開始レベル | 理由 |
| --- | --- | --- |
| SEO調査 | L0-L1 | 公開前確認があれば安全に始めやすい |
| 議事録整理 | L0-L1 | タスク登録前に担当者確認が必要 |
| 問い合わせ分類 | L0-L1 | 返信送信は外部送信になる |
| 営業フォロー | L0-L1 | 顧客文面とCRM更新は承認が必要 |
| 会計月次 | L0-L1 | 仕訳登録や請求は高リスク |
| 労務入社手続き | L0-L1 | 個人情報と雇用条件を扱う |
| 契約レビュー | L0-L1 | 法的判断をAIだけで確定しない |
| WordPress保守 | L0-L1 | 本番変更はL4以上 |
| RPA実行 | L1 | 実行先がレガシーシステムでも結果は本番操作になる |

## 実行前確認テンプレート

```text
次の操作は実行前承認が必要です。

- 対象サービス:
- 操作種別:
- 対象IDまたはURL:
- 件数:
- 変更前:
- 変更後:
- 失敗時の戻し方:
- 承認者:

「承認します」と明示されるまで実行しないでください。
```

## APIキー / トークン方針

- 管理者トークンをAIエージェントに渡さない。
- 読み取り専用トークンから始める。
- 書き込み権限は業務ごとに分ける。
- トークンはプロンプトや議事録に貼らない。
- 外部送信できるエージェントに個人情報トークンを同時に持たせない。

## OSSテンプレートでの表記ルール

- テンプレートには「許可すること」「禁止すること」「承認チェック」を入れる。
- 外部送信、登録、更新、削除、支払い、契約、人事、インフラ変更は必ず承認チェックを入れる。
- 「AIが判断してよい」と書かず、「AIは候補を出す」「人間が確定する」と書く。
