# Codex、Claude Code、Cursor、Claude Desktopの使い分け

## Codex

リポジトリの調査、コード編集、テスト実行、Issue対応、ドキュメント整備などに向いています。既存コードの読み取りと変更を一つの作業として扱いたいときに便利です。

## Claude Code

ターミナル上でコードベースを読み、計画、編集、テスト、Git操作を進める用途に向いています。`CLAUDE.md` にプロジェクト固有のルールを書くと、作業の再現性が上がります。

## Cursor

エディタ内でのリアルタイム編集や補完、既存ファイルを見ながらの手作業に向いています。人間が主導しながら素早く直す場面で使いやすいです。

## Claude Desktop

MCP経由で外部サービスと会話しながら使う用途に向いています。コード編集よりも、資料確認、外部ツール連携、日常業務の補助に向くことがあります。

## 選び方

- リポジトリを一気に調べて修正したい: Codex または Claude Code
- エディタで人間が細かく触りたい: Cursor
- MCPツールを会話で使いたい: Claude Desktop
- 高リスク業務を扱う: どのツールでも人間承認を必須にする
