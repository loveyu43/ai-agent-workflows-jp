# WordPressサーバー保守ユースケース

## 目的

WordPressサイトの状態確認、軽微な保守、障害調査の準備をAIエージェントに補助させます。

## AIに任せる範囲

- SSL状態の確認
- PHPバージョン確認
- エラーログの確認
- WordPressインストール状況の確認
- 改善案の作成
- 作業手順書の作成

## 人間承認が必要な範囲

- WordPress削除
- DNS変更
- DB削除
- PHPバージョン変更
- Cron変更
- メールアカウント変更
- 本番反映

## 推奨ワークフロー

1. AIに「確認だけ」を依頼する
2. 現状、問題点、危険度、推奨対応を出させる
3. 変更が必要な場合は影響範囲と戻し方を出させる
4. 人間が承認する
5. 実行後にログと結果を保存する
