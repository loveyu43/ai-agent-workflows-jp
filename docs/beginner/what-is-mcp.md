# MCPとは何か

MCPはModel Context Protocolの略で、AIアシスタントが外部ツールやサービスと接続するための仕組みです。

従来は人間が管理画面やAPIを直接操作していた作業を、AIが「道具」として呼び出せるようにします。たとえば、サーバー情報の取得、ファイル検索、GitHubのIssue確認、会計データの取得などです。

## 便利な点

- AIに状況確認や作業案の作成を任せやすくなる
- 複数のサービスをまたいだ作業を一つの会話で扱いやすくなる
- Codex、Claude Code、Cursorなどの開発支援ツールから業務ツールを扱える可能性が広がる

## 注意する点

MCPは便利ですが、接続先の権限が強いほどリスクも大きくなります。DNS、SSL、データベース、請求、顧客情報、メールなどを扱う場合は、AIが何を実行できるのかを必ず確認してください。

## 基本方針

- 最初は読み取り専用で使う
- 変更、削除、発行、送信は人間の確認後にする
- APIキーをチャットや公開リポジトリに貼らない
- 本番環境の前にテスト環境で試す
- ログを残す
