# Roadmap

## Next 30 Days Maintenance Plan

These are the concrete maintainer tasks planned for the next 30 days after `v0.1.3`.

- Verify `last_checked` dates and official URLs for all 50 Japanese MCP/API provider records.
- Add at least 10 provider-specific notes for high-risk Japanese APIs such as accounting, contracts, payments, HR, EC, and infrastructure.
- Expand the evaluation dataset from 30 to 45 records with more failure-mode prompts for external sending, billing, contract review, and infrastructure changes.
- Add 5 beginner-friendly `good first issue` tasks that can be completed by editing one template, one provider note, or one dataset record.
- Add 3 `help wanted` tasks for source verification, Japanese workflow review, and Codex/Claude Code behavior comparisons.
- Publish a short `v0.2.0` release candidate checklist covering validation, source review, and unsafe-action wording.
- Review all new templates against the approval-boundary matrix before release.
